Why Testing Matters

  • Improves code quality by catching bugs early
  • Increases developer confidence in code changes
  • Facilitates safe refactoring
  • Essential for scalable applications
  • Reduces time spent on debugging

Testing Types

  • Unit Tests: Test individual functions or components in isolation
  • Integration Tests: Test interactions between multiple components
  • End-to-End Tests: Test the entire application flow from user perspective

Tools Overview

  • Jest: A JavaScript testing framework developed by Facebook. Great for unit and integration tests with built-in mocking and assertion library.
  • React Testing Library: A testing utility for React that encourages testing components from the user's perspective, not implementation details.

Unit Testing with Jest

Example: Testing transaction validation logic

// validateTransaction.js
function validateTransaction(amount, balance) {
  if (amount <= 0) return false;
  if (amount > balance) return false;
  return true;
}

module.exports = validateTransaction;

// validateTransaction.test.js
const validateTransaction = require('./validateTransaction');

describe('validateTransaction', () => {
  it('should return false for negative amount', () => {
    expect(validateTransaction(-100, 1000)).toBe(false);
  });

  it('should return false for amount exceeding balance', () => {
    expect(validateTransaction(1500, 1000)).toBe(false);
  });

  it('should return true for valid amount', () => {
    expect(validateTransaction(500, 1000)).toBe(true);
  });
});

Testing React Components with React Testing Library

Example: Testing the Amount Input Component

// AmountInput.js
import React from 'react';

function AmountInput({ value, onChange }) {
  return (
    <label>
      Transfer Amount:
      <input
        type="number"
        value={value}
        onChange={onChange}
        min="0"
      />
    </label>
  );
}

export default AmountInput;

// AmountInput.test.js
import { render, screen, fireEvent } from '@testing-library/react';
import AmountInput from './AmountInput';

test('renders amount input with label', () => {
  render(<AmountInput value="" onChange={() => {}} />);
  const input = screen.getByLabelText(/transfer amount/i);
  expect(input).toBeInTheDocument();
});

test('calls onChange when input value changes', () => {
  const handleChange = jest.fn();
  render(<AmountInput value="" onChange={handleChange} />);
  const input = screen.getByLabelText(/transfer amount/i);
  fireEvent.change(input, { target: { value: '500' } });
  expect(handleChange).toHaveBeenCalledTimes(1);
});

Integration Testing

Testing the complete transaction flow with mocked dependencies.

Example: Testing the transfer process that combines validation and API call.

// transferService.js
const validateTransaction = require('./validateTransaction');

async function processTransfer(amount, balance, recipientId) {
  if (!validateTransaction(amount, balance)) {
    throw new Error('Invalid transaction');
  }
  // Mock API call
  const response = await fetch('/api/transfer', {
    method: 'POST',
    body: JSON.stringify({ amount, recipientId })
  });
  return response.json();
}

module.exports = processTransfer;

Use Jest to mock the fetch and test the integration.
